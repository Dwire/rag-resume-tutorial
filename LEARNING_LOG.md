## Step 0 — Foundations & planning (2026-07-09)

**What was built:** No code (by design). Created the project's memory and foundation documents:
- `CLAUDE.md` (teaching contract + session-start protocol)
- `docs/01-technology-stack.md`, `docs/02-architecture.md`, `docs/03-learning-syllabus.md`
- `docs/decisions/0001`–`0004` (initial ADRs)

**Key decisions (see ADRs for full rationale):**
1. **No orchestration framework (no LangChain/LlamaIndex)** — we hand-roll chunking, retrieval, and prompt assembly so every moving part is understood and explainable in interviews (ADR-0001).
2. **Local embeddings via sentence-transformers** — Anthropic has no embeddings API; a local open-source model is free, offline, and forces us to learn what embeddings actually are (ADR-0002).
3. **ChromaDB as the vector store** — the standard beginner-friendly local vector DB; persistent, pip-installable, cosine similarity built in (ADR-0003).
4. **Claude API (claude-opus-4-8) for generation** — generation quality matters for resumes; the grounding/anti-hallucination work happens in our prompt construction, which we own (ADR-0004).

**Concepts covered:** the RAG pipeline at a high level (ingest → chunk → embed → store → retrieve → prompt → generate), why RAG exists (grounding LLM output in your own documents), framework vs. hand-rolled trade-off.

**Plan revision (same day, student-approved):** compared our plan against an alternative outline (generated by Sonnet) and adopted its three genuinely better ideas:
1. **Vector store built twice** — hand-rolled NumPy store gets its own step (4), used for real retrieval (5), then ChromaDB swapped in behind a shared `VectorStore` interface with a side-by-side comparison (6). See ADR-0005.
2. **Generation split** into resume (Step 8, structure constraints) and cover letter (Step 9, length constraints) — consistent with the "small steps" contract.
3. **pytest added** from Step 1 onward for deterministic stages; LLM stages get validators/evals instead (rule added to CLAUDE.md as contract item 7).
Kept from our plan (defended): multi-query retrieval as the core of Step 5, explicit model choice with ADR, ADR/interview-relevance documentation depth. Setup and document loading merged into a single Step 1 to keep the syllabus at 13 steps (0–12).
